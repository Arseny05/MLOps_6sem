Данный проект посвящен созданию системы ML позволяющей предсказывать спрос на прокат велосипедоов. Данные для обучения были взяты из датасета [London bike sharing dataset](https://www.kaggle.com/datasets/hmavrodiev/london-bike-sharing-dataset)
Проект разделен логически на 2 части: обработчик сырых данных и модель, предсказывающая спрос.
I. Обработчик сырых данных.
  Обработчик сырых данных - программа, принимающая на вход необработанные данные, выдающая на выходе очищенные, закодированные для ML модели. Обработчик имеет вид конвейера, который опираясь на модули имеет 6 логических частей:
  * Обработчик потоковых данных (stream.py) - модуль, принимающий на вход батчи сырых данных, помещающий их в базу данных, подсчитывая статистику по пропускам из каждого батча. Обработчик может менять режимы работы в зависимости от настроек конфигурационного файла gathering.json.
  * Программа, оценивающая качество данных (quality.py) - модуль, подключающийся к базе данных database.db, предоставляющий информацию о качестве сырах данных. Информация записывается в файл data_quality.json.
  * EDA - Jupyter Notebook, проводящий экспедиционный анализ данных. Выявляет зависимости между данными, а также проводит статистический анализ, рассматривая логнормальную модель.
  * Обработчик пропущенных значений - выполняет импутацию пропусков на основе алгоритма MissForest - итеративный алгоритм импутирующий пропущенные данные за счет итеративного пострения случайного леса.
  * Регулятор дрифта данных - программа, которая на основе тестов Колмогорова-Смирнова и хи-квадрат извлекает дрифт между данными. Следует отметить, что за счет изменения времен года паремтрыЮ связанные с погодой претерпевают дрифт, поэтому куда критичнее здесь не столько дрифт, сколько его динамика.
  * Кодировщик данных - программа, кодирующая категориальные переменные и обрабатывающая числовые. Применяется one-hot, label и тригонометрическое кодирования. Для числовых переменных используется MinMaxScaler и StandartScaler.
  Все данные хранятся в базе данных databade.db, содержащей 7 таблиц:
  1. batches - таблица, регистрирующая пришедшие батчи в БД
  2. clean - импутированные данные после применния алгоритма MissForest
  3. metaparameters - таблица, содержащая данные по пропускам в каждом батче по каждой категории
  4. rents - таблица, хранящая сырые данные
  5. sqlite_sentence - системная таблица БД
  6. var_1, var_2 - таблицы, подающиеся модели ML. Отличаются разными кодировками для категориальных и трансформерами для числовых данных


Система ML моделей, предсказывающих спрос, состоит из 6 основных логических модулей:

1) nn_model.py - содержит в себе классы MyDataSet и MyModel, реализующие объекты набора данных и нейросетевого типа моделей, имеющего фиксированную архитектуру - 4 блока (Linear, LeakyReLU, BatchNorm1d и Dropout) и финальный слой Linear, в системе соответственно. Гиперпараметры модели - learning_rate (lr), число эпох (epoch), размерность входа (in_features), weight_decay и процент dropout'а на слоях (dropout). Класс MyModel обладает методами forward (реализующим прямой проход по слоям), fit (реализующим цикл обучения модели), save_ и load_checkpoint (реализующими соответственно сохранение и загрузку файлов .pt, являющихся весами для модели)

2) monitor.py - содержит в себе класс Monitor, осуществляющий мониторинг использования памяти и времени выполнения процесса. Данный класс имеет гиперпараметр sample_interval, отвечающий за интервалы записи метрик, и методы start (запускающий мониторинг), stop (останавливающий мониторинг), sample_memory (осуществляющий мониторинг, сохряняющий метрики) и summary (возвращающий основные метрики).

3) Orchestrator.py - содержит класс ModelOrchestrator, являющийся контроллером моделей и осуществляющий взаимодействие с базой данных, представленной в файле models_monitoring.db. Основные аттрибуты класса: MODELS - словарь моделей, обладающий минимальной информацией для обучения и применения моделей - индекс модели, её тип, путь к чекпойнту, DATASETS - список кортежей вида (X, y), являющихся наборами данных, загруженными в систему, config - файл конфигурации, загружающийся с запуском программы из пути "./config.yaml", data_dir - путь к папке для сохранения отчетов, графиков и файлов датасетов, загруженных в систему (.csv), ckp_folder - путь к папке для сохранения файлов чекпойнтов моделей, загруженных в систему, CUR_DS - выбранный для применения модели датасет. Основные методы класса: get_model_overview, get_dataset_overview - возвращают основную информацию о модели или датасете с индексом index, register_dataset - добавляет датасет из пути data_path и поддерживает расширения .csv, .json и .db (для .db обязателен параметр table - название таблицы в файле), append_to_dataset - добавляет к датасету index данные из пути data_path при условии совпадения колонок, dataset_deletion - удаляет датасет index из системы, model_creation - создает модель одного из типов ("nn", "sgd", "tree", "for") - нейросеть MyModel, StochasticGradientDescent из scikit-learn, DecisionTreeRegressor или RandomForestRegressor из scikit-learn соответственно с параметрами kwargs, model_deletion - удаляет модель index из системы, check_for_drift - проверяет модель index на дрифт по всем различным датасетам, записи о которых есть в models_monitoring.db, выдавая свой вердикт пользователю (вероятность дрифта + совет по дальнейшим действиям), print_model_info - выводит информацию обо всех чекпойнтах модели, select_model_version - выбирает путь к чекпойнту index (нумерация из таблицы из print_model_info), fit_model - обучает модель index на выбранном датасете с параметрами anew (обучение с нуля или дообучение) и save_metrics (сохранение графика использования памяти (RAM для моделей из sklearn, GPU + RAM для MyModel)), predict - делает предсказание моделью index на выбранном датасете, update_model - проводит Cross-Validation для модели index (или для всех - параметр "all") на текущем датасете, save_model - сохраняет модель index, explain_model - на примере случайной строки из текущего датасета показывает feature_importances и путь к решению модели (для tree и forest), auto_select_and_predict - в зависимости от входных данных считает некоторые эвристики и выбирает наиболее релевантную модель и делает ей предсказание

4) checker.py - содержит класс BackgroundChecker, раз в monitor_inverval секунд проверяет все зарегистрированные модели на дрифт по эвристикам.

5) shell.py - содержит класс Shell, осуществляющий в цикле взаимодействие с пользователем. Список команд доступен командой help или ?. После этого при команде help <название команды> можно узнать синтаксис и смысл команды

6) main.py - модуль для запуска работы с проектом. Вызов "python main.py" откроет интерфейс командной строки

Дополнительные файлы, необходимые для работы системы:

1) models_monitoring.db - создаётся автоматически при запуске системы в первый раз, но также используется для восстановления работоспособности системы при аварийной обстановке. Восстанавливаются все модели и датасеты, зарегистрированные в системе в момент аварийной остановки

2) config.yaml - содержит в себе все гиперпараметры для работы системы.
