Данный проект посвящен созданию системы ML позволяющей предсказывать спрос на прокат велосипедоов. Данные для обучения были взяты из датасета [London bike sharing dataset](https://www.kaggle.com/datasets/hmavrodiev/london-bike-sharing-dataset)
Проект разделен логически на 2 части: обработчик сырых данных и модель, предсказывающая спрос.
I. Обработчик сырых данных.
  Обработчик сырых данных - программа, принимающая на вход необработанные данные, выдающая на выходе очищенные, закодированные для ML модели. Обработчик имеет вид конвейера, который опираясь на модули имеет 6 логических частей:
  * Обработчик потоковых данных (stream.py) - модуль, принимающий на вход батчи сырых данных, помещающий их в базу данных, подсчитывая статистику по пропускам из каждого батча. Обработчик может менять режимы работы в зависимости от настроек конфигурационного файла gathering.json.
  * Программа, оценивающая качество данных (quality.py) - модуль, подключающийся к базе данных database.db, предоставляющий информацию о качестве сырах данных. Информация записывается в файл data_quality.json.
  * EDA - Jupyter Notebook, проводящий экспедиционный анализ данных. Выявляет зависимости между данными, а также проводит статистический анализ, рассматривая логнормальную модель.
  * Обработчик пропущенных значений - выполняет импутацию пропусков на основе алгоритма MissForest - итеративный алгоритм импутирующий пропущенные данные за счет итеративного пострения случайного леса.
  * Регулятор дрифта данных - программа, которая на основе тестов Колмогорова-Смирнова и хи-квадрат извлекает дрифт между данными. Следует отметить, что за счет изменения времен года паремтрыЮ связанные с погодой претерпевают дрифт, поэтому куда критичнее здесь не столько дрифт, сколько его динамика.
  * Кодировщик данных - программа, кодирующая категориальные переменные и обрабатывающая числовые. Применяется one-hot, label и тригонометрическое кодирования. Для числовых переменных используется MinMaxScaler и StandartScaler.
  Все данные хранятся в базе данных databade.db, содержащей 7 таблиц:
  1. batches - таблица, регистрирующая пришедшие батчи в БД
  2. clean - импутированные данные после применния алгоритма MissForest
  3. metaparameters - таблица, содержащая данные по пропускам в каждом батче по каждой категории
  4. rents - таблица, хранящая сырые данные
  5. sqlite_sentence - системная таблица БД
  6. var_1, var_2 - таблицы, подающиеся модели ML. Отличаются разными кодировками для категориальных и трансформерами для числовых данных
